# G143 Options Surface Evidence v1

Status: implemented backend evidence and feature contract

Date: 2026-07-20

## Purpose

G143 closes the provider-to-feature contract gap identified by the Market State Intelligence reconciliation. Massive bounded option snapshot records now carry enough selected quote, contract, and request lineage to produce the option observations required by research hypotheses H001, H004, H006, and H007.

The gate remains hypothesis-driven and storage-bounded. Transient candidates are used for compact distributions, while at most seven deterministic source contracts are persisted for one symbol/session.

## Implemented Scope

- Parses Massive `last_quote` bid, ask, and nanosecond timestamp; contract exercise style and shares per contract; and per-page provider request ID.
- Preserves provider request IDs, pages fetched, and pagination completeness in capture metrics. A bounded page stop is explicitly `provider_candidate_window_incomplete`, not silently complete.
- Adds migration `000033_marketops_options_surface_evidence` for nullable quote fields plus selection cell, policy version, and score on selected contract rows.
- Uses selection policy `marketops.options.surface_selection.v1` for seven cells: 30/60/90-DTE ATM plus 30/60-DTE 25-delta put and call.
- Keeps the existing provider DTE, moneyness, candidate, page, and symbol limits. Non-selected candidates remain transient.
- Aggregates moneyness, expiration, DTE, and absolute-delta OI/volume buckets from the bounded transient candidate set.
- Records eligible candidate count and explicit rejection counts for missing IV, delta, spot, invalid DTE, nonstandard contract size, and nonstandard exercise style.
- Extends the MarketOps v1 catalog to 29 definitions and 39 deterministic observation slots per state.
- Emits seven IV cells, 30/60 and 60/90 term slopes, 30/60 25-delta risk reversals, and aggregate surface-selection confidence.
- Emits real put/call 30-DTE 25-delta midpoint, extrinsic premium, premium-to-spot, and spread observations from positive non-crossed quotes.
- Emits put/call 30-DTE 25-delta `oi_change_1d` with exact current and prior selected-contract lineage.
- Keeps missing, invalid, stale, partial, and warning states explicit. Missing quotes and OI are never converted to numeric zero.
- Exposes quote, contract, request, and selection lineage additively through the existing options-chain API DTO.

## Hypothesis Compatibility

The G138 contract is now generated by the production materializer:

- H001 receives 30/60-DTE 25-delta put IV, 30-DTE put extrinsic premium, dimensioned put OI change, transitions, and coverage.
- H004 receives 30/60/90 ATM IV and persistent generated transitions.
- H006 receives put/call 30-DTE 25-delta midpoint and IV observations plus transitions.
- H007 receives dimensioned put/call 30-DTE 25-delta OI change and transitions.

A Massive-shaped HTTP fixture passes through the real adapter, chain conversion, seven-cell selector, G137 materializer, and G138 evaluator. All four research hypotheses become eligible without direct feature-observation injection.

## Quality Semantics

- `usable`: required source values are present and point-in-time valid.
- `usable_with_warning`: a numeric quote calculation is valid but quote timestamp or selection-policy metadata is unavailable.
- `stale`: quote timestamp is not from the materialized session. The numeric value remains auditable but is not admitted by hypotheses.
- `invalid`: bid/ask is non-positive or crossed.
- `missing`: a required selected contract, quote side, spot, current OI, or prior comparable surface cell is absent.

Pagination completeness describes the bounded provider candidate window. It does not change a complete seven-cell surface into bulk-chain persistence and does not use contract count as a readiness proxy.

## Storage Boundary

For a prospective symbol/session:

1. Resolve canonical same-session equity close.
2. Fetch only the configured provider DTE/strike window within request and candidate limits.
3. Build compact distributions from transient candidates.
4. Select at most seven source contracts.
5. Persist only selected rows plus one compact distribution and capture metrics.
6. Materialize versioned feature observations from persisted selected evidence.

G143 does not store a complete provider chain, synthesize historical quotes, enable a scheduler, broaden calculation beyond explicit symbols, promote research, materialize production signals, or add frontend behavior.

## Validation

- Focused Massive adapter, options ingestion/readiness/distribution, state, hypothesis, history-runner, coverage-runner, PostgreSQL, and API tests pass.
- Quote tests cover present, missing, crossed, and stale markets.
- Selection tests verify seven rows, stable cell/policy/score lineage, and rejection of irrelevant contracts.
- Distribution tests verify transient DTE/delta OI-volume buckets and rejection counts.
- State tests verify term slope, risk reversal, selection confidence, premium, dimensioned OI change, full 39-slot lineage, and stable identities.
- The Massive-shaped end-to-end acceptance test makes H001/H004/H006/H007 eligible from generated production observations.
- Migration `000033` applied and rolled back in a disposable PostgreSQL database; all eight columns were present after up and absent after down.

No live Massive request was required for deterministic acceptance. Live prospective sessions still depend on same-session canonical equity evidence and remain subject to G142 request budgets.
